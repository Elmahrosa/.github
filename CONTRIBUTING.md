# Contributing to Elmahrosa

Thank you for contributing. These repositories prioritize **security**, **auditability**, and **sovereign-grade reliability**.

## Contribution Rules
1. Keep changes small and reviewable.
2. Follow existing module boundaries and naming conventions.
3. Do not introduce dependencies without justification.
4. No secrets in code. Use `.env.example`.

## Workflow
1. Fork the repo and create a feature branch.
2. Run:
   - `npm test` (or project-specific test command)
   - `npm run lint` if present
3. Open a PR with:
   - What/Why
   - Risk assessment
   - Test evidence

## Security & Responsible Disclosure
If you find vulnerabilities, do **not** open a public issue. Use `SECURITY.md`.
